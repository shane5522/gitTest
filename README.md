## Commit Message Convention

형식: `타입: 한 줄 요약`

### 타입
- feat: 새 기능
- fix: 버그 수정
- docs: 문서(README 등)
- style: 포맷/공백/세미콜론 (로직 변경 X)
- refactor: 리팩토링
- test: 테스트 코드
- chore: 설정, 빌드, 잡일

#### 추가
- perf: 성능 개선
- build: 빌드 설정
- ci: CI 설정

### 한 줄 요약 규칙
1. 현재형으로 작성
2. 50자 이내
3. 마침표 사용 ❌
4. 왜가 아니라 무엇을 했는지 작성

### 예시
- feat: 회원 가입 기능 추가
- fix: 로그인 실패 시 예외 처리 오류 수정
- docs: README에 개발 환경 설명 추가
- style: 들여쓰기 및 코드 포맷 정리
- refactor: 게시글 서비스 로직 분리
- test: 회원 서비스 단위 테스트 추가
- chore: gitignore 및 빌드 설정 정리

#### 추가 예시
- perf: 게시글 조회 쿼리 성능 개선
- build: Gradle 의존성 버전 업데이트
- ci: GitHub Actions 빌드 파이프라인 추가
